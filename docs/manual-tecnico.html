1. Descripción general

Moldes Fácil Tiler es una herramienta de escritorio de uso interno para el equipo de producción de Moldes Fácil. Su función principal es convertir archivos PDF de tamaño plotter (anchos y altos) en hojas del tamaño estándar que usan las clientas para imprimir en casa (A4 o Carta), y subirlos automáticamente al sitio web con la información correcta del catálogo.

Problema que resuelve

Los moldes de costura se diseñan en formato plotter (ej.: 90×120 cm). Las clientas imprimen en hojas A4 o Carta que deben ensamblarse. El Tiler divide automáticamente el PDF grande en la cantidad exacta de hojas necesarias, manteniendo escala 1:1, y agrega marcas de registro y etiquetas de cuadrícula (A1, B2…) para facilitar el ensamblaje.

Flujo principal

  1. El operador arrastra las carpetas con PDFs plotter al programa.
  2. La app detecta dimensiones y calcula el grid (ej.: 4×6 = 24 hojas A4).
  3. El operador ajusta el offset del grid si es necesario (para evitar cortar zonas críticas del molde).
  4. Hace clic en "Cargar al sitio": la app tilea los PDFs, los sube a DigitalOcean Spaces y los asocia al producto correspondiente en el catálogo web.
  5. Todo queda registrado automáticamente en Google Sheets.

3. Arquitectura del sistema

Procesos de Electron

Electron divide la app en dos procesos separados que se comunican por IPC (Inter-Process Communication):

ProcesoArchivoResponsabilidad
Mainmain.jsAcceso a disco, red, S3, Google Sheets, diálogos del sistema, worker pool
Rendererrenderer/app.jsUI, estado visual, interacción del usuario
Preloadpreload.jsPuente seguro entre main y renderer (contextIsolation)
Workerstiler/pdfWorker.jsProcesamiento pesado de PDFs (4 instancias en paralelo)

5. Convenciones de nombres de archivos

La app soporta dos formatos de nombre para los PDFs de entrada:

Formato A — Talla prefijo

{talla_prefijo} molde {código}.pdf

Ejemplos:
  T-14 molde 569.pdf    →  código: 569,  talla: 14
  T-12 molde 443A.pdf   →  código: 443A, talla: 12

Formato B — Código prefijo

{código} TALLA {talla}.pdf

Ejemplos:
  370A TALLA L.pdf      →  código: 370A, talla: L
  684 TALLA M.pdf       →  código: 684,  talla: M

Archivos de salida (tileados)

{nombre_original} A4.pdf
{nombre_original} Carta.pdf
{nombre_original} Plotter.pdf

Ejemplo de salida:
  T-14 molde 569 A4.pdf
  T-14 molde 569 Carta.pdf
⚠ Importante: Los archivos cuyo nombre no coincida con ninguno de los dos formatos mostrarán la etiqueta "sin código" en la tabla y no podrán subirse al sitio web. Verificar los nombres antes de procesar lotes grandes.

6. SOPs — Procedimientos operativos estándar

SOP-01 — Iniciar sesión

  1. Abrir la aplicación Moldes Fácil Tiler.
  2. Ingresar el correo: info@moldesfacil.cl
  3. Ingresar la clave de acceso y hacer clic en "Ingresar".
  4. La sesión se recuerda por 30 días. No es necesario repetir este paso en cada uso.

SOP-02 — Cargar moldes al sitio web (flujo principal)

  1. Arrastrar la carpeta de moldes a la zona de drop, o usar "Seleccionar carpeta de moldes". Se pueden seleccionar múltiples carpetas a la vez.
  2. Esperar a que la app detecte las dimensiones de cada archivo (indicador de progreso).
  3. Verificar los badges de validación junto a cada archivo:
    ✓ verde = código encontrado en el catálogo
    ⚠ amarillo = sin código (nombre no reconocido)
    ✗ rojo = código no encontrado en el catálogo
  4. Seleccionar los formatos a generar: A4, Carta y/o Plotter.
  5. Hacer clic en "Cargar al sitio".
  6. Esperar a que el progreso llegue al 100%. La barra muestra archivo por archivo.
  7. Al terminar aparece el resumen: archivos subidos OK y errores (si los hay).
  8. Si hay errores, usar el botón "Reintentar archivos con error" o descargar el reporte .txt.

SOP-03 — Ajustar el grid de un molde

  1. En la tabla de archivos, hacer clic en el botón "👁 A4" o "👁 Carta" del archivo a ajustar.
  2. En el modal de previsualización, usar los sliders ↔ (horizontal) y ↕ (vertical) para desplazar el grid.
  3. Alternativamente, hacer clic en "✨ Auto-ajustar" para que la app detecte automáticamente el mejor offset.
  4. Una vez conforme, hacer clic en "💾 Guardar ajuste para este molde".
  5. Repetir para cada formato (A4 / Carta) si es necesario.

SOP-04 — Descargar PDFs localmente (sin subir al sitio)

  1. Cargar los archivos siguiendo SOP-02 pasos 1–4.
  2. Seleccionar una carpeta de destino con el botón "Elegir…".
  3. Usar "⬇ Descargar seleccionados" o "⬇ Descargar todos".
  4. Los PDFs se guardan en subcarpetas por formato: NombreCarpeta A4/, NombreCarpeta Carta/.

SOP-05 — Cerrar sesión

Hacer clic en el botón "Cerrar sesión" en la esquina superior derecha. La sesión también se cierra automáticamente tras 15 minutos de inactividad (con aviso de 60 segundos previo).

SOP-06 — Revisar historial de cargas

  1. En el panel derecho, expandir "📋 Historial de cargas".
  2. Ver las últimas 20 sesiones con fecha, hora, archivos subidos y errores.
  3. Si aparece el indicador "⏳ registros pendientes", hacer clic en "Sincronizar ahora" para enviar los datos pendientes a Google Sheets.

Comportamiento offline

Si no hay conexión a internet al momento de la operación:

  • Los datos se guardan localmente en sheets-queue.json (carpeta de datos del usuario).
  • Al recuperar la conexión, la app sincroniza automáticamente (intento cada 5 minutos).

10. Build y distribución

macOS (desarrollo y producción)

# Iniciar en modo desarrollo
env -u ELECTRON_RUN_AS_NODE npm start

# Generar instalador .dmg para macOS
npm run build:mac
⚠ Importante: El comando npm start debe ejecutarse con env -u ELECTRON_RUN_AS_NODE para evitar el error "app is undefined".

Windows (automático via GitHub Actions)

  1. Ir al repositorio en GitHub → pestaña Actions.
  2. Seleccionar el workflow "Build Windows".
  3. Hacer clic en "Run workflow".
  4. Esperar ~5 minutos. El instalador .exe aparece como artifact descargable.

El workflow (.github/workflows/build-win.yml) corre en un runner windows-latest con Node 18 y genera el ejecutable con electron-builder.
